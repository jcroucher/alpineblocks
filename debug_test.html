Alpine Blocks - Nested Block Class Name Debug Test

Debug Instructions

Step 1: Open your browser's Developer Console (F12 ‚Üí Console tab)
Step 2: Load your Alpine Blocks editor with a Columns block
Step 3: Drag and drop a block (e.g., Paragraph, Header) into a column
Step 4: Watch the console for debug output with these prefixes:
  • === COLUMN DROP DEBUG START === - Shows what happens during drop
  • === INITBLOCK DEBUG START === - Shows block creation in Editor
  • === RENDER NESTED BLOCKS DEBUG START === - Shows rendering process
  • === GET TOOL INSTANCE DEBUG START === - Shows tool instance creation
  • === SERIALIZE BLOCK CONFIG DEBUG START === - Shows serialization process
  • === BLOCKS JSON DEBUG START === - Shows final JSON export
Step 5: Try exporting the blocks as JSON to trigger serialization
What to Look For: Find where block.class changes from the proper class name (e.g., "Paragraph") to "Object"

Key Debug Points to Monitor

  1. Initial Drop: Check if blockData.class is correct when dropped
  2. After initBlock: Verify nestedBlock.class is preserved
  3. After Adding to Column: Check if class is still correct in column array
  4. During Rendering: See if class remains correct during render cycles
  5. During Serialization: Identify where class becomes "Object"

Expected Flow

The debug logs will show you the exact moment when the class property changes from the intended class name to "Object". This typically happens due to:

  • JSON serialization/deserialization losing the class property
  • Object cloning without preserving the class
  • Alpine.js reactive updates that modify the object
  • A merge operation that doesn't preserve the class property
